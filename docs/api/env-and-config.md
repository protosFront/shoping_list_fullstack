# API env and config

## Правило разделения
- `infra/.env` - только для Docker/инфры
- `apps/api/.env` - только для API

## Файлы
- `apps/api/.env.example` - коммитится, шаблон переменных
- `apps/api/.env` - локально, не коммитится

## Создание локального env
Из корня репозитория:
```bash
cp apps/api/.env.example apps/api/.env
```
Дальше меняем значения под локальную машину.

## Переменные (минимум для M1.3)
- `API_HOST`
- `API_PORT`
- `CORS_ORIGIN`

## Поведение при старте
- API загружает `apps/api/.env` при старте.
- Если переменной нет или она некорректна, процесс завершится с понятным сообщением.
