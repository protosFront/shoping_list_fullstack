# Env files & GitIgnore

## Смысл
- `.env.example` — коммитится, это **шаблон/контракт** (какие переменные нужны)
- `.env` — **локальный файл** с реальными значениями (пароли/ключи), **не коммитится**

Цель: любой разработчик мог скопировать пример и запустить проект.

---

## Файлы
- `infra/.env.example` — в репозитории
- `infra/.env` — локально, в `.gitignore`

---

## Создание локального env
Из корня репозитория:

```bash
cp infra/.env.example infra/.env

- в ручную меняем данные в infra/.env
